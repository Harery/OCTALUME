## üìö The 8 Phases: Your Journey from Idea to Production

Each phase builds on the last, with clear entry criteria, defined deliverables, and quality gates ensuring you're ready before moving forward.

<details>
<summary><strong>üìñ See all 8 phases at a glance</strong></summary>

| Phase | What You'll Do | Why It Matters | Duration | Owner |
|:-----:|----------------|---------------|----------|-------|
| **1** | Define what & why | Avoid building the wrong thing | 4-6 weeks | Product Owner |
| **2** | Specify requirements | Prevent scope creep & ambiguity | 4-8 weeks | Product Owner |
| **3** | Design architecture | Make technical decisions upfront | 6-10 weeks | CTA |
| **4** | Plan the work | Know what, who, and when | 2-4 weeks | Project Manager |
| **5** | Build the software | Agile sprints, working software | Variable | Tech Lead |
| **6** | Validate quality & security | Catch issues before production | 4-8 weeks | QA Lead |
| **7** | Deploy to production | Smooth, controlled releases | 1-2 weeks | DevOps |
| **8** | Operate & improve | Keep it running & getting better | Ongoing | SRE |

</details>

<details>
<summary><strong>üîç Deep dive: Each phase explained</strong></summary>

### Phase 1: Vision & Strategy

**The problem you solve:** Teams often dive into building without clarity on *why* they're building it and *what success looks like*.

**What you'll create:**
- **Business Case:** Why this project matters to the business
- **PRD (Product Requirements Document):** What exactly you're building
- **Success Metrics:** How you'll measure achievement

**Outcome:** Executive buy-in, budget approval, and crystal-clear product vision.

### Phase 2: Requirements & Scope

**The problem you solve:** Vague requirements lead to rework, delays, and frustration.

**What you'll create:**
- **Requirements List:** Every feature, clearly defined
- **Traceability Matrix:** Link requirements to business goals
- **Acceptance Criteria:** Know when each requirement is met

**Outcome:** Clear scope that prevents scope creep while ensuring nothing critical is missed.

### Phase 3: Architecture & Design

**The problem you solve:** Technical decisions made reactively create technical debt.

**What you'll create:**
- **System Architecture:** How all components fit together
- **Technology Choices:** What tech stack and why
- **Threat Models:** Security designed in, not bolted on

**Outcome:** A technical blueprint that guides implementation with confidence.

### Phase 4: Development Planning

**The problem you solve:** "We'll figure it out as we go" doesn't work at enterprise scale.

**What you'll create:**
- **Work Breakdown Structure (WBS):** Every task identified
- **Resource Plan:** Who does what, when
- **Sprint Plan:** 2-week sprints with clear goals

**Outcome:** A realistic timeline with proper resource allocation.

### Phase 5: Development Execution

**The problem you solve:** Long development cycles without feedback lead to wasted effort.

**What you'll approach:** Agile sprints with:
- 2-week sprint cadence
- Daily standups (15 minutes)
- Sprint reviews with stakeholders
- Retrospectives for continuous improvement

**Outcome:** Working software delivered incrementally with constant feedback.

### Phase 6: Quality & Security Validation

**The problem you solve:** Finding bugs in production is expensive and dangerous.

**What you'll validate:**
- **Functional Testing:** Does it work as specified?
- **Security Testing:** Is it secure?
- **Performance Testing:** Is it fast enough?
- **UAT:** Do actual users accept it?

**Outcome:** Confidence that your software is production-ready.

### Phase 7: Deployment & Release

**The problem you solve:** Deployments often fail due to poor preparation.

**What you'll ensure:**
- **Deployment Plan:** Step-by-step deployment process
- **Rollback Plan:** What if something goes wrong?
- **Smoke Tests:** Quick verification of success

**Outcome:** Smooth, controlled production releases.

### Phase 8: Operations & Maintenance

**The problem you solve:** Software needs ongoing care after launch.

**What you'll provide:**
- **Monitoring:** Track health and performance
- **Incident Response:** Handle issues quickly
- **Continuous Improvement:** Regular updates and enhancements

**Outcome:** Reliable, stable production operations.

</details>

## üîß Installation

### Prerequisites

**Before we begin, let's verify your system is ready:**

```bash
# Check Node.js version (need 18+)
node --version
# Expected output: v18.x.x or higher
# If you see an older version, upgrade at: https://nodejs.org

# Check npm version (need 9+)
npm --version
# Expected output: 9.x.x or higher
# Older? Update Node.js and npm will update too

# Check git version (need 2.0+)
git --version
# Expected output: git version 2.x.x or higher
# Not installed? Get it at: https://git-scm.com
```

**Minimum requirements:**
- Node.js 18.0.0+ | npm 9.0.0+ | git 2.0+
- Linux: Ubuntu 20.04+, Debian 11+, CentOS 8+, or compatible

### Setup (Takes 3 Minutes)

```bash
# 1. Copy OCTALUME somewhere convenient
cp -r OCTALUME/ ~/projects/

# 2. Install MCP server dependencies
cd ~/projects/OCTALUME/.claude/mcp-server
npm install

# 3. Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# 4. Verify installation
cd ~/projects/OCTALUME
claude --version
# Expected output: Claude Code version info
```

**üéâ You're ready!** Start Claude Code from your OCTALUME directory and the framework loads automatically.

## üîó Traceability: Tracking Everything

Every artifact follows the format: **`P{N}-{SECTION}-###`**

**Examples:**
- `P1-VISION-001` ‚Äî Phase 1, Vision document, item 1
- `P3-ARCH-042` ‚Äî Phase 3, Architecture, item 42
- `P5-CODE-789` ‚Äî Phase 5, Code commit, item 789

**The traceability chain:**

```
Epic ‚Üí Feature ‚Üí Story ‚Üí Commit ‚Üí Build ‚Üí Artifact ‚Üí Release ‚Üí Test ‚Üí Result
```

**Why this matters:** When something goes wrong (or right!), you can trace it back to the original decision, requirement, or stakeholder request.

## üìã Common Issues: We've Got You Covered

<details>
<summary><strong>Claude Code not loading OCTALUME?</strong></summary>

**Symptom:** CLAUDE.md content not available when you start Claude Code

**Most likely cause:** You're not in the OCTALUME directory

**Quick fix:**
```bash
# Check where you are
pwd
# Should show: /path/to/OCTALUME

# If not, navigate there
cd /path/to/OCTALUME

# Try again
claude
```

**Still not working?** Make sure the `.claude` directory exists:
```bash
ls -la .claude/
# You should see: CLAUDE.md, ORCHESTRATOR.md, etc.
```

**If `.claude` is missing:** Re-copy OCTALUME including hidden files:
```bash
cp -r /source/OCTALUME/ /destination/OCTALUME/
```

</details>

<details>
<summary><strong>MCP tools not appearing in Claude Code?</strong></summary>

**Symptom:** MCP server tools don't show up in Claude Code

**Most likely cause:** Dependencies not installed correctly

**Quick fix:**
```bash
cd .claude/mcp-server
rm -rf node_modules package-lock.json
npm install
cd ../..
```

**Verify installation:**
```bash
ls .claude/mcp-server/node_modules/
# You should see a list of installed packages
```

**Still failing?** Check Node.js version:
```bash
node --version
# Must be 18+ for MCP server compatibility
```

</details>

<details>
<summary><strong>Can't move to the next phase?</strong></summary>

**Symptom:** Phase transition blocked

**Most likely cause:** Exit criteria not met for current phase

**What to do:**
```bash
# In Claude Code, ask:
"We've completed Phase [X]. Validate exit criteria and run go/no-go."
```

**Claude will:**
1. Check all exit criteria for your current phase
2. Verify entry criteria for the next phase
3. Identify any gaps or blockers
4. Help you resolve issues
5. Run the go/no-go decision once ready

**Common blockers:**
- Missing deliverables ‚Üí Complete them
- Unsigned approvals ‚Üí Get signatures
- Failed quality gates ‚Üí Fix and retest
- Incomplete documentation ‚Üí Update docs

</details>

## üéì Methodology: Best of Both Worlds

- Go/No-Go decisions

- Daily standups

**Why this hybrid works:** Upfront planning and architecture prevent expensive mistakes, while Agile execution gives you flexibility and rapid feedback during development.
